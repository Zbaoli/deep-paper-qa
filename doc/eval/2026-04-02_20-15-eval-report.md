## 四、问题发现

### 重要：冗余调用仍然严重

**影响范围**: 30 次连续重复调用（同一工具连续调用 2+ 次）
**高发区域**: content 题和 mixed 题
**典型案例**:
- Q19（6 次调用）：search_abstracts 搜了 3 次不同关键词，vector_search 搜了 2 次，还多调了 execute_sql
- Q24（7 次调用）：search_abstracts 搜了 4 次，vector_search 1 次，execute_sql 2 次
**根因**: prompt 中的效率规则对 DeepSeek 约束力不足，Agent 仍然倾向"多搜几次确认"

### 重要：mixed 题缺少 execute_sql

**影响范围**: Q27、Q33（2/13 = 15.4%）
**表现**: Agent 直接用 search_abstracts + vector_search 搜内容，跳过了统计筛选步骤
**根因**: 当问题中"引用超过 50"或"AAAI 2024"等筛选条件可以通过 search_abstracts 的 where 参数实现时，Agent 选择跳过 execute_sql

### 改进：Q38 评测逻辑

**影响**: 1 题
**表现**: CVPR 2026 不存在于数据库，Agent 直接根据 schema 中的枚举值回答"无法查询"，行为合理但评测判定为失败
**根因**: 评测要求 sql 类型必须调用 execute_sql，但 Agent 的直接回答其实更好

## 五、优化建议

### 建议 1：强化效率规则为硬性约束

影响：30 次冗余调用，Q19（6 次）、Q24（7 次）
预期：content 题平均调用数从 3.4 降到 2.0，mixed 题从 3.8 降到 3.0

```diff
--- a/src/deep_paper_qa/prompts.py
+++ b/src/deep_paper_qa/prompts.py
 效率规则：
+- 单个问题最多调用 4 次工具，超过此限制必须基于已有信息回答
 - 优先用一次工具调用获取足够信息，不要反复用不同关键词搜索同一主题
 - 如果第一次搜索已返回 5+ 条相关结果，直接基于这些结果回答，不要再搜
-- 对同一问题，不要同时用 search_abstracts 和 vector_search 搜索相似内容，选一个即可
+- 禁止对同一问题同时用 search_abstracts 和 vector_search，选一个即可
+- 禁止连续调用同一工具 2 次以上
 - 混合问题最多 2-3 次工具调用：先统计再内容，不要反复验证
 - 如果 3 次工具调用后仍无满意结果，基于已有信息回答并说明局限
```

### 建议 2：强化 mixed 题路由规则

影响：Q27、Q33 缺少 execute_sql
预期：mixed 路由正确率从 84.6% 提升到 92%+

```diff
--- a/src/deep_paper_qa/prompts.py
+++ b/src/deep_paper_qa/prompts.py
 工具选择规则：
 - 统计/计数/排名/趋势问题 → execute_sql（用全文检索做主题过滤）
 - 精确关键词/术语搜索 → search_abstracts（返回论文列表+摘要片段）
 - 模糊/概念性搜索 → vector_search（基于语义相似度）
-- 混合问题 → 先 execute_sql 做统计，再 search_abstracts 或 vector_search 查内容
+- 混合问题（同时涉及统计和内容）→ **必须**先用 execute_sql 做统计/筛选，再用检索工具查内容。不要跳过 execute_sql。
 - 对比分析 → 分别查询后综合回答
 - 如果某个工具不可用，降级到其他工具
```

### 建议 3：Q38 评测逻辑修正

影响：Q38 CVPR 2026
预期：sql 路由正确率恢复到 100%

```diff
--- a/eval/run_eval.py
+++ b/eval/run_eval.py
         if q_type == "sql":
-            tool_correct = "execute_sql" in called_set
+            # Agent 直接回答（0 次调用）且回答中包含"无法查询"/"不存在"也视为正确
+            tool_correct = "execute_sql" in called_set or (
+                len(called_set) == 0 and any(
+                    kw in final_answer for kw in ["无法查询", "不存在", "没有", "不包含"]
+                )
+            )
```
