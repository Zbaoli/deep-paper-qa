## 四、问题发现

### 致命：content 题质量分偏低（平均 3.12/5）

**影响范围**: 27 道 content 题中 11 道 < 3.0
**根因**: Agent 的搜索策略倾向于"广撒网"——多轮用不同关键词搜索，但综合回答质量不高。搜索结果多≠回答好。
**典型案例**:
- Q11 "RAG chunking 策略"：4 次调用，质量仅 2.0
- Q15 "attention 优化方法"：4 次调用，质量仅 2.0
- Q43 "GNN 不要综述"：1 次调用但质量 2.0（路由正确但回答不够深入）

### 重要：mixed 题仍有 17.6% 缺少 execute_sql

**影响范围**: Q22、Q24、Q34（3/17）
**表现**: Agent 用 search_abstracts 的 where 参数替代 execute_sql 做筛选
**与上次对比**: Q22、Q24 持续出现，Q33 本次修复，Q34 新增失败
**根因**: prompt 中"混合问题 → 先 execute_sql 做统计"的约束力不够，Agent 倾向于选择最简路径

### 重要：冗余调用问题加剧

**影响范围**: 11 道题 >4 次调用（占 16.9%），最高 Q48 达 8 次
**高发区域**:
- content 平均 3.5 次/题（vs 上次 3.4）
- mixed 平均 4.1 次/题（vs 上次 3.9）
**模式分析**:
  1. 同一工具连续调用 2-3 次换不同关键词（Q48: 5 次 execute_sql + 2 次 search_abstracts）
  2. content 题额外调用 execute_sql 统计（Q37, Q39, Q48）
  3. mixed 题 execute_sql 多次重试拼 SQL（Q36: 6 次 execute_sql, Q64: 4 次 execute_sql）

### 改进建议：评测逻辑容错

**Q38** "CVPR 2026"：Agent 知道数据范围直接回答，质量 4.75 但评测判错
**Q52** "数据库有哪些字段"：Agent 从 prompt 知识回答，质量 5.0 但评测判错
**Q53** "asdfghjkl"：Agent 识别为无意义输入，质量 3.5 但评测判错

### 观察：线上日志印证冗余问题

线上 15 个独立查询中：
- "Bengio + Hinton 合著" 触发 **9 次** execute_sql
- "ICLR RAG 趋势" 一个会话触发 **18 次** 工具调用
- "各会议论文数量" 简单查询也触发 3-4 次调用
- 冗余问题在线上比评测更严重（无 recursion_limit 约束）

```diff
+++ b/eval/run_eval.py

     if q_type == "sql":
-        tool_correct = "execute_sql" in called_set
+        # Agent 直接回答（0 次调用）且回答合理也视为正确

+        )
     elif q_type == "mixed":
```

预期：消除 3 个误判，正确率提升到 95.4%

### 建议 2：prompt 中对效率施加硬限制

影响：11 道题 >4 次调用（Q48=8次, Q23=7次, Q36/Q40/Q64/Q28=6次）
预期：content 平均调用从 3.5 降到 2.5，mixed 从 4.1 降到 3.0

```diff
--- a/src/deep_paper_qa/prompts.py
+++ b/src/deep_paper_qa/prompts.py
@@ -132,6 +132,9 @@
 效率规则：
+- **硬限制：单个问题最多调用 4 次工具**。超过 4 次必须立即基于已有信息回答，不得继续调用。
+- 禁止连续用同一工具搜索 2 次以上（换关键词也算）。如果第一次搜索无结果，最多再试一次换表述，然后必须结束。
+- 纯内容理解问题（"XX 是什么"/"XX 怎么工作"）不要调 execute_sql，只用检索工具。
 - 优先用一次工具调用获取足够信息，不要反复用不同关键词搜索同一主题
```
