## 路由架构

用一个路由节点将用户问题分为六类，分别走不同 pipeline：

```
路由节点（LLM 六分类）
  ├─ 拒答 → 模板回复
  ├─ 普通问题 → ReAct subgraph
  ├─ 深度研究 → DeepResearch subgraph
  ├─ 论文精读 → PaperReading subgraph
  ├─ 论文对比 → PaperCompare subgraph
  └─ 趋势分析 → TrendAnalysis subgraph
```

### 1. 拒答

非 AI 论文相关问题，直接模板回复，零 token 消耗。

示例：
- "今天天气怎么样？"
- "帮我写一段 Python 排序代码"
- "帮我润色一下我的论文摘要"

### 2. 普通问题（ReAct）

单次或少量检索即可回答的问题，由 ReAct Agent 自行决定工具调用。

示例：
- "2024年NeurIPS收录了多少篇论文？"
- "有哪些关于 Chain-of-Thought prompting 的论文？"
- "2024年引用最高的 RAG 论文讲了什么？"
- "Hinton 近两年发了什么论文？"
- "ICLR 2023 关于 RLHF 的论文有几篇，分别讲了什么？"

### 3. 深度研究（DeepResearch）

需要多主题/多角度分步研究的复杂问题，输出结构化研究报告。

流程：澄清追问（最多 3 轮）→ 研究简报 → 分步检索 + 压缩（循环）→ 报告生成

示例：
- "调研 AI for Science 在蛋白质结构预测方向的最新进展"
- "总结 2023-2025 年 LLM Agent 的研究脉络，包括关键论文和技术路线"
- "梳理 text-to-image 从 GAN 到 diffusion 的技术演进，给出综述报告"
- "写一份关于多模态大模型的研究综述，覆盖主要方法和代表性工作"

### 4. 论文精读（PaperReading）

输入一篇特定论文，输出逐章节解读。需要全文 PDF 支持。

示例：
- "帮我精读 Attention Is All You Need 这篇论文"
- "详细解读这篇 FlashAttention 论文的方法部分"
- "这篇 DPO 论文的核心贡献和实验设计是什么？"

### 5. 论文对比（PaperCompare）

输入两篇或多篇论文，多维度对齐对比。需要全文 PDF 支持。

流程：识别对比维度 → 分别提取信息 → 语义对齐 → 结构化对比输出

示例：
- "对比 DPO 和 RLHF 这两篇论文的方法差异"
- "FlashAttention v1 和 v2 有什么改进？"
- "对比 RAG 和 long context 两种方案的优劣"

### 6. 趋势分析（TrendAnalysis）

时间维度的统计 + 内容演进分析，数据驱动。

工具侧重：execute_sql（按年统计）+ search_abstracts（各阶段代表作）

示例：
- "RAG 近三年的发展趋势"
- "知识蒸馏这个方向是在升温还是降温？"
- "2020-2025 年 GAN 相关论文的数量变化和研究重心转移"
- "Transformer 架构改进的研究热度变化"

## 分类判断标准

| Pipeline | 核心判断 |
|----------|---------|
| 拒答 | 与 AI 论文无关 |
| 普通问题 | 1-4 次工具调用可回答 |
| 深度研究 | 可拆解为 3+ 个子问题，需要结构化报告 |
| 论文精读 | 针对一篇特定论文的深入解读 |
| 论文对比 | 针对两篇+论文的多维度对比 |
| 趋势分析 | 时间维度的统计趋势 + 各阶段代表作 |

## 实现优先级

### P0（当前数据即可支撑）

- **路由节点** — LLM 六分类，是整个架构的入口
- **拒答** — 模板回复，实现成本最低
- **普通问题（ReAct）** — 改造现有 Agent，从 prompt 中剥离深度研究逻辑

### P1（当前数据即可支撑，流程较复杂）

- **深度研究（DeepResearch）** — 从现有 prompt 约束改为 LangGraph 显式 subgraph（澄清 → 规划 → 分步检索压缩 → 报告生成）
- **趋势分析（TrendAnalysis）** — 现有 execute_sql + search_abstracts 组合即可，需要设计固定的"统计 → 分段检索代表作 → 趋势总结"流程

### P2（依赖全文 PDF 接入）

- **论文精读（PaperReading）** — 需要 PDF 下载解析能力，暂返回"功能开发中"
- **论文对比（PaperCompare）** — 需要全文支持多维度对比，暂返回"功能开发中"

### 边界 case 注意

路由容易混淆的问题，需在路由 prompt 中给出判断示例：
- "RAG 近三年有哪些重要论文" → **普通问题**（不需要时间趋势统计）
- "RAG 近三年的发展趋势" → **趋势分析**（需要按年统计 + 各阶段代表作）
- "总结 RAG 领域的研究脉络" → **深度研究**（需要多角度分步研究 + 综述报告）
- "对比 RAG 和 long context 的研究趋势" → **趋势分析**（核心是时间维度对比，不是论文级对比）
